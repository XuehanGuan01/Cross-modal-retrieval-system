# 项目名称：交互式跨模态检索 Agent（Iterative Cross-modal Retrieval Agent）

## 1. 项目目标
构建一个支持多轮对话、跨模态（图像 ↔ 文本）检索的 Agent 系统。用户可通过上传图片或自然语言描述，在多个垂直领域（医疗、电商、植物、动物等）中找到目标对象。

## 2. 核心功能需求
- **前端**：提供按领域分类的检索入口（医疗 / 电商 / 植物 / 动物 /…），支持图片上传 + 文本对话。
- **检索流程**：
  1. 用户输入（图片或描述）。
  2. 系统执行跨模态检索，返回 Top-5 相似结果及其 caption。
  3. 将 Top-5 caption 作为上下文，通过 RAG 注入 LLM。
  4. LLM 推理输出最可能的 Top-3 结果，并附带置信度或解释。
  5. 返回对应的图片和简短结论。
- **多轮对话**：Agent 可基于历史检索结果和用户反馈，迭代优化检索条件（例如调整特征、排除错误项）。

## 3. 数据集来源
> 当前由开发者自行搜集各领域数据，集成公开跨模态数据集，数据集保证有实际图片+图片描述caption放入metadata。

## 4. 技术栈（当前 + 计划）
| 阶段   | 技术栈                                                      |
| ---- | -------------------------------------------------------- |
| 基础实现 | 跨模态模型（CLIP）、FAISS 或 Milvus 进行向量检索、FastAPI 后端、Vue+vite 前端 |
| 后续增强 | LangChain（链式调用）、LangGraph（多步检索 + 反射）、RAG（检索增强生成）、对话状态管理  |

## 5. 系统迭代路线图（建议）
### Phase 1：基础跨模态检索
- 实现单一领域的图像 ↔ 文本检索（如商品）。
- 返回 Top-5 caption，前端展示图片 + 文本。

### Phase 2：多领域 + 前端分类
- 支持医疗 / 电商 / 植物 / 动物 四个领域。
- 前端的检索入口按领域隔离（不同领域的向量库分开检索）。

### Phase 3：对话式检索（第一版 Agent）
- 封装检索为 LangChain Tool。
- 加入简单对话：用户问“有更小的红色款吗？”→ Agent 重新检索 + 过滤。

### Phase 4：LangGraph 多步推理
- 设计 Graph：理解意图 → 跨模态检索 → RAG 增强 → 反思检索结果 → 输出 Top-3。
- 支持分支（如检索结果置信度低时追问用户）。

### Phase 5：完整交互式 Agent
- 长期记忆（跨会话偏好）。
- 主动建议（例如“根据你之前查的植物，这朵可能是某某”）。
